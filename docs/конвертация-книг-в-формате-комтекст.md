# Конвертация книг в формате Комтекст

**11.03.2026**

Формат электронных книг должен позволять конвертировать файл в любой другой формат. Автор оформляет книгу в одном формате, остальные создаются автоматически. Этот принцип реализован во всех архивах на сайте [comtext.space](https://comtext.space).

Конвертация происходит [при сборке сайта](https://github.com/comtextspace/comtext_builder) через [Пандок](https://pandoc.org). Сейчас из формата Комтекст генерируются файлы ФБ2 и ЕПАБ.

Пандок [генерирует файлы ФБ2 с ошибками](./проблемы-конвертации-маркдаун-в-фб2-через-пандок.md). Генератор сайта исправляет их постобработкой файла. Валидные файлы создаются, если заполнить [поля метаинформации](./format-comtext.md): заголовок, автор, язык и жанр. Дополнительно можно указать аннотацию и обложку. Полученные файлы без ошибок загружаются в онлайн-библиотеки, основанные на ФБ2, [например в coollib.in](https://coollib.in/a/267246-valtraut-fritsevna-shelike).

Распространение конвертированных файлов снижает риск их потери, даже если основной сайт закроется. Недостаток децентрализации в том, что появляется множество версий одной книги с разным количеством опечаток. Хотя сейчас в ФБ2 добавляются специальные поля `id` и `version` для точного сравнения файлов, они не решают всех проблем.

Предстоит ещё много работы по улучшению конвертации и добавлению новых форматов, но в целом схема работает. Основные технологии: Гит, облегчённый формат разметки, генератор статических сайтов и Пандок.
